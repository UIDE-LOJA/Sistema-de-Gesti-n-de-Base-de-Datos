Esta actividad de trabajo autónomo te permitirá analizar y documentar decisiones de optimización de bases de datos mediante la elaboración de un informe técnico. Te enfocarás en registrar métricas de rendimiento, justificar decisiones de indexación, y documentar el impacto de las optimizaciones aplicadas.

¿Qué realizarás?

Componente Instrucciones específicas Criterios de calidad
Contexto del caso Describe el caso de estudio: tipo de aplicación, volumen de datos, consultas críticas. Incluye esquema de base de datos relevante. Contexto claro y relevante.
Métricas baseline Documenta métricas iniciales de las 3 consultas: latencia (ms), rows procesados, cost, plan de ejecución. Usa EXPLAIN ANALYZE. Métricas precisas y completas.
Análisis de planes Analiza planes de ejecución identificando: tipo de scan, operaciones costosas, cuellos de botella. Explica por qué son ineficientes. Análisis técnico fundamentado.
Decisiones de índices Justifica qué índices crear: tipo (B-Tree, Hash, GiST), columnas, razón técnica. Considera trade-offs (espacio, writes). Decisiones bien justificadas.
Métricas post-optimización Documenta métricas después de crear índices. Calcula mejora porcentual en latencia, reducción de rows, cambio en cost. Mejoras cuantificadas claramente.
Conclusiones Breve análisis (máximo 150 palabras) sobre lecciones aprendidas, cuándo usar índices, y recomendaciones para casos similares. Conclusiones fundamentadas.

Estructura del informe:

Sección 1: Contexto y Baseline

  • Descripción del caso de estudio
  • Esquema de tablas relevantes
  • Consultas críticas identificadas
  • Métricas baseline (tabla con: consulta, latencia, rows, cost, plan)

Sección 2: Análisis y Optimización

  • Análisis de cada plan de ejecución
  • Identificación de cuellos de botella
  • Decisión de índices con justificación técnica
  • Comandos SQL de creación de índices

Sección 3: Resultados y Conclusiones

  • Tabla comparativa: antes vs después
  • Cálculo de mejoras (%, factor de mejora)
  • Gráfico comparativo (opcional pero recomendado)
  • Lecciones aprendidas y recomendaciones

Ejemplo de tabla comparativa:

Consulta Latencia Antes Latencia Después Mejora Índice Creado
SELECT con WHERE 89.67 ms 0.067 ms 1337x idx_orders_customer
JOIN 2 tablas 234.5 ms 12.3 ms 19x idx_products_id, idx_orders_product
GROUP BY agregación 456.8 ms 45.2 ms 10x idx_sales_date_category

Consideraciones para decisiones de índices:

  • Cuándo crear índice: Columnas en WHERE, JOIN, ORDER BY frecuentes
  • Tipo de índice: B-Tree (general), Hash (igualdad), GiST (geoespacial)
  • Trade-offs: Mejora lecturas pero ralentiza escrituras, consume espacio
  • Índices compuestos: Orden de columnas importa (selectividad)

Materiales: Presentación Semana 4, documentación PostgreSQL sobre EXPLAIN, guías de optimización. Herramientas: PostgreSQL, pgAdmin, Excel para tablas.
